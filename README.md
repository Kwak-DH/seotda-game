# 섯다 (Seotda) - 인디 카드게임 프로젝트

섯다 변형 규칙(10장, 1~10월 각 1장) 엔진 + 웹 UI로 만드는 인디 카드게임.
11, 12월 패는 존재하지 않으며, 광패는 1·3·8월 세 장뿐이다.
추후 로그라이크 성장 요소 / 맞고 확장 / 온라인 대전을 붙일 수 있도록
**게임 로직(core)과 UI를 완전히 분리**한 구조로 설계했습니다.

## 프로젝트 구조

```
seotda-game/
├── packages/
│   ├── core/     # 순수 TypeScript 게임 로직 (프레임워크 독립적)
│   └── ui/       # React + Vite 프론트엔드
├── pnpm-workspace.yaml
└── package.json
```

## core 패키지 현황 (완료)

- **카드**: 10장 덱 생성 (1~10월 각 1장, 광패는 1·3·8월뿐) (`Card.ts`, `Deck.ts`)
- **족보 판정**: 광땡(38 > 18 > 13) > 땡 > 알리 > 독사 > 구삥 > 장삥 > 장사 > 세륙 > 끗수 > 망통
  (`HandRank.ts`, `HandComparator.ts`)
- **게임 상태/베팅**: 콜/다이/레이즈, **올인 처리**, 팟 관리, 승자 판정, 칩 지급
  (`GameEngine.ts`, `Bet.ts`, `Deal.ts`)
- **기본 AI**: 족보 강도 기반 규칙형 AI, 약한 패에서도 낮은 확률로 블러핑 (`BasicAI.ts`)
- **테스트**: 43개 유닛/통합 테스트 전부 통과 (족보 전 케이스 + 베팅 흐름 + 올인 + 칩 보존 검증)
- **대규모 시뮬레이션 검증**: AI 대 AI로 1000판+ 자동 플레이하여 무한루프/칩 불일치 없음을 확인

## ui 패키지 현황 (완료 - 1차 버전)

- React + Vite, `useGameEngine` 훅으로 core 엔진을 상태에 연결
- 화투 카드 컴포넌트, 베팅 패널(콜/다이/레이즈, 올인 시 라벨 자동 전환), 승부 결과 오버레이
- 사람 vs 기본 AI로 실제 플레이 가능한 최소 기능 완성

## 시작하기

```bash
pnpm install

# 테스트 실행
pnpm test

# core 빌드
pnpm build:core

# UI 개발 서버 실행 (http://localhost:5173)
pnpm dev:ui
```

## 알려진 이슈 / 설계 메모

- **올인 처리**: 초기 구현에서 한쪽이 칩 부족으로 콜하면 베팅 라운드가 상대방 응수 전에
  종료되어버리는 버그가 있었음. `hasActed` 플래그를 플레이어별로 추적하여 "이번 라운드에
  모든 활성(비올인) 플레이어가 최소 한 번 행동했는지"까지 확인하도록 수정. 레이즈 발생 시
  다른 플레이어들의 `hasActed`를 리셋하여 재대응을 요구하도록 처리함.
- AI의 레이즈가 칩 부족으로 막히면 콜(올인)로, 그마저 안 되면 다이로 자동 폴백하는 안전장치가
  `useGameEngine` 훅에 있음. 사람 플레이어는 UI에서 애초에 불가능한 액션 버튼이 비활성화됨.

## 설계 원칙

- `core`는 UI 프레임워크를 전혀 모릅니다. 모든 함수는 순수 함수(입력 → 출력)로,
  상태 변경 없이 새 상태를 반환합니다. 테스트와 디버깅이 쉽고, 나중에 서버 사이드
  (온라인 대전)로 그대로 재사용 가능합니다.
- 족보 판정처럼 실수하면 게임 신뢰도가 무너지는 로직은 반드시 유닛테스트로
  모든 케이스를 검증합니다.
- 유닛테스트로 못 잡는 "여러 액션이 겹쳐서 나는 상태 버그"(예: 올인 타이밍)는
  대규모 자동 시뮬레이션(AI vs AI 1000판)으로 추가 검증합니다.
